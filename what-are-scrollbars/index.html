What are Scrollbars?

Question: What happens on a webpage when the text is too long for the window?

Answer: The window becomes scrollable.

Question: As the page is scrolled, what usually appears near the window edge?

Answer: A scrollbar.

Question: What does the scrollbar do?

Answer:

Scrollbars show the current position of where the webpage is scrolled.

Scrollbars also have behaviors.

Scrollbars can be interactive. A finger or cursor can usually drag them to scroll more rapidly thru a page. Pushing the spacebar usually scrolls the page forward.

Scrollbars can be manipulated. There are JavaScript APIs to get and set the scroll position There are CSS rules to change how they are styled.

Question: Are scrollbars available to more than the window?

Answer: Yes. Almost any area in a webpage can do this.

Question: What element needs to be written to do this?

Answer: No element is written to do this. It is generated automatically, or manually with CSS.

🤨

Question: If it has an interface, API, and can be styled, then why is it not written as an element?

Answer: It’s not written as an element because it’s not content, and because it’s conditional.

Question: What is an element that is conditional and not written out?

Answer:

This kind of element is an affordance.

An affordance is how something makes itself clear how to use it.

Question: Are there other affordances than scrollbars in webpages?

Answer: Yes. Most webpages use more affordances than just scrollbars.

Question: What is another common affordance?

Answer: Most responsive websites use a ‘hamburger’ affordance to navigate menus on small screens.
